Data Cleaning & Transformation at Scale

Worked with a real building-sensor dataset from a multi-year Berkeley research study — a genuinely messy real-world database with billions of readings, inconsistent metadata, and OCR errors baked into the source tables. Cleaned it up using entity resolution, outlier detection, missing-data imputation, and ontology roll-up queries — all in raw Postgres SQL, no pandas shortcuts.

PostgreSQL · CTEs · Window Functions · Recursive Queries
Outlier detection & Winsorization (Hampel X84 method)
CREATE VIEW cleaned_data AS
SELECT
    time, id, value, median, mad, is_outlier,
    CASE
        WHEN mad = 0 THEN value
        WHEN NOT is_outlier THEN value
        WHEN value > median
            THEN median + 3 * 1.4826 * mad
        ELSE
            median - 3 * 1.4826 * mad
    END AS clean_value
FROM labeled_data;

Ontology roll-up: counting sensors by category (recursive graph query)
WITH sensor_children AS (
    SELECT subject AS sensor_child
    FROM ontology
    WHERE object = 'Brick#Sensor'
        AND predicate = 'subClassOf'
)
SELECT sc.sensor_child, COUNT(DISTINCT md.id) AS count
FROM metadata AS md
INNER JOIN mapping AS map ON md.class = map.rawname
INNER JOIN transitive_subClassOf AS t ON map.brickclass = t.subject
INNER JOIN sensor_children AS sc ON t.object = sc.sensor_child
GROUP BY sc.sensor_child
ORDER BY sc.sensor_child;
